Website Mapping

Mapping Description

The website mapping section has four primary mapping types for content classification (to run/execute on server or send to client browser). The fifth, for Sub Mapping, is different in that it's matched after the Proxy and FQDN mapping stage but before the website content. See 'Getting Started' and 'Request Handling' for more details on that process. Out of the four main mapping rulesets for content, they are matched in the following order (first rule wins). There can be multiple mappings per mapping section that are ordered by longest URI path first so that those paths are matched before the shorter path. Make sure to not use same URI path names in each section as one may win over the subsequent rules.
