Sub Mapping

'Porject Website Sub Mapping' under the 'Website Mapping' section is used to map a URI path to another website in the project. This is not used for mapping to websites in a different project. Depending on your upstream configuration for how a domain is resolved to the server or servers, you would potentially configure a 'Proxy Map' under the 'Resolve' panel to map a domain and path to a project and website, then use sub mapping to map a URI path of your website to another website.

In these examples it defines where in the URL / URI path sub mapping is used. From a VHost preview path, a Proxy Map or an DNS FQDN Map, the sub map URI path is noted in bold letters.

https:// hostname/vhost/project::website/sub_map/*
https:// www.domain.com/proxy_path/sub_map/*
https:// www.domain.com/sub_map/*

Create a Sub Map

The way to create a sub mapping from the website to another website, start by clicking the website you want to add a sub map to, and click the add icon to bring up the dialog.




Enter the URI path that will map to a website, select the website in the project that it will map to and click 'Map Path' button.
