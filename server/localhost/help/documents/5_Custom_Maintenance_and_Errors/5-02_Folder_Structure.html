Folder Structure

Basic Configuration

There is a default folder structure to the '_maintenance_page' and '_error_pages'. For the most part they are static content. For a simple example, the following diagram just shows the singular files in these folders to match the website general settings. The HTML files can be simple text or have an HTML layout with embedded CSS styles and JavaScript. API Response files can be simple JSON string. The helper buttons on the website settings panel will create a basic layout like this.




Advanced Configuration

You can have a more advanced setup with these pages. Creating a sub folder named 'api' and placing API files in here will be treated as a server side execute function (similar to API Dynamic Path Mapping in the Website Configuration section). The API folder might have some appeal for things like having a maintenance page with a notice to users for expected downtime or status. There is a Default System website template called 'simple' which has this basic functionality with a 'message.txt' file in the root of the maintenance page folder (see below). The only file that will execute server side is the 'message.js' file in the 'api' folder. Also note that additional folders for CSS styles and JavaScript files to be sent to client browser. See next section, 'Special URI Paths' for the contents of the HTML files for how to ensure these load properly.
