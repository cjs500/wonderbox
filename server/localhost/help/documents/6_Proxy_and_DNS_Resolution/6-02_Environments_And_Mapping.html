Environments and Mapping

It's important to understand multiple environments and DNS resolution to server IP, Proxy/Load Balancers, or local Hosts file when you start working with mapping. Reference to 'Getting Started' and 'Environments', that is one hypothetical layout for different environments (below). It doesn't mean you will have your environments matching this layout. It could be much more simple or leverage Kubernetes instead of traditional VM servers with external load balancers, etc. For sake of simplicity, a lot of these examples will use traditional environments to help describe mapping the system software can support.




Simple Environment

Before getting into this example, from a security perspective, it will most likely not look like this. This is just a functional example. Your environment will most likely have a DMZ or multiple, API gateway, potentially a local code repo, VPNs or users may not work on code from home even (or use virtual desktop for remote access). Also, separate database services from web layer, etc.

What the environment could look like, is the following. Generally speaking, leveraging a GitHub repo and working within you own branch of a project or website, and later can be merged into the development environment in the company. Also, a project or website can be cloned in the system which can allow a manager of a project to focus each project clone to a different repo branch. A cloned project will not include the Proxy or DNS mapping settings of the master project but a can still be viewed with a VHost preview (or create Proxy Mapping to resolve a subfolder of a domain name to the clone). At some point all the changes can be merged into the main branch and pulled into the master project. The local HOSTS file can allow you to similate the company internal DNS and the 'Proxy Map' or 'DNS FQDN' mapping can work even on your local workstation as it would in the company network with local DNS. If your workstation is placed on the company network, you would need to remove the HOSTS file entry to resolve to the internal DNS server.
