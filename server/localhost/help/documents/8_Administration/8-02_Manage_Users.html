Manage Users

User management and access control to project and website settings can be configured from this panel. From a fresh install, the only user is the 'admin'. At the current time the user management is local to this Management UI. There is no ties in with SSO or LDAP services at the current time. Current user management is good for small groups or simple access control to login to the Management UI.


Create a User Account

Start with creating a user from the add icon to bring up the add user dialog.




At minimum fill in the user and password, the 'Display Name' and 'Email Address' are optional.




The user account will add to the list.




User Account Locked or Disable

Users may accidentally lock themselves out. You can unlock their account from the main user list. By default the user gets 3 attempts and will lock the account for 30 minutes.

If you want to disable a user account you can from the main user list.




Manage User Permissions

There are two general permission types, global permissions and project specific permissions as noted below.

Global permissions:

  • admns - Full access to everything
  • Project Admin - Full access to all projects
  • Project Create - Create new projects, clone other projects with at least read only, full management of created prjects
  • Template Admin - Allow add and delete of templates
  • DNS Admin - Allow add, delete and modify of Proxy and DNS FQDN Mapping

Project Access:

  • Admin - Full access to everything in the project
  • Settings - Project settings like enable / disable, description, etc
  • Website Admin - Full access to add, clone, remove, rename websites and all of the settings
  • Website Settings - Website settings only and re-assignment of website for Proxy and DNS FQDN Mapping
  • Files Admin - Full Access to managing files from Managment UI
  • Files Read - Read Access to files from Managment UI
  • Read Only - Read Access to project panels

To set these permissions, click the arrow icon to the right of the user. The first tab will open to 'Global Permissions'. If you want to give users access to global permissions, you can simply check the boxes on this tab.




The 'Project Access' panel are where you want to set more granual access to a project. You may want to grant users read access to a project and more access to others. Click the arrow icon to the right of the project you want to grant permissions and simply click the boxes on this tab.




Users don't need to log out and back on to activate the settings. As soon as the user refreshed from the browser refresh or the project menu above the project tree navigation it will reload the user permissions. This can also be done when removing permissions from a user.
